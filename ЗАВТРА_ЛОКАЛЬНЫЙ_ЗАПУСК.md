## 1. Env один раз

```bash
cp .env.example .env
```

В `.env` для локалки обязательно проверь порты compose:

```
POSTGRES_HOST=localhost
POSTGRES_PORT=5435
REDIS_URL=redis://localhost:6382/0
MINIO_ENDPOINT=http://localhost:9010
API_BASE_URL=http://localhost:8000
SELLER_PUBLIC_URL=http://localhost:3000
ADMIN_VPN_REQUIRED=false
ADMIN_2FA_REQUIRED=false
CLOUD_API_MOCK=1
WORKER_DEPLOY=local
WORKER_PIPELINE_MODE=stub
WORKER_TOKEN=worker-dev-token
```

Без ЮKassa платежи не пройдут — ключи можно оставить пустыми до интеграции.  
Для облака без ключа: `CLOUD_API_MOCK=1`. Для реального API — токен из `Адреса Облачных Воркеров.txt`.

Seller/admin API URL:

```bash
# apps/web-seller — обычно .env.local
NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1

# apps/web-admin — .env
VITE_API_URL=http://localhost:8000/api/v1
```

## 2. Инфраструктура (Docker)

Только storage (рекомендуется для разработки API/UI на хосте):

```bash
docker compose up -d postgres redis minio clickhouse
```

Порты:

| Сервис | Host |
|--------|------|
| Postgres | `localhost:5435` |
| Redis | `localhost:6382` |
| MinIO API | `localhost:9010` |
| MinIO Console | `localhost:9011` (minioadmin / minioadmin) |
| ClickHouse | `localhost:8123` |

Опционально весь стек в Docker (оркестратор + celery + prometheus + grafana):

```bash
docker compose up -d
# API: http://localhost:8000
# Grafana: http://localhost:3003 (admin/admin)
```

HA-кластер (не для повседневной разработки):

```bash
docker compose -f docker-compose.ha.yml up -d
# см. docs/deployment/HA.md
```

Остановка:

```bash
docker compose down
# с данными: docker compose down -v
```

## 3. Оркестратор (API)

```bash
cd backend/orchestrator
py -3 -m venv .venv

# Windows
.venv\Scripts\activate
# Linux/macOS
# source .venv/bin/activate

pip install -e ".[dev]"
alembic upgrade head
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Проверка: http://localhost:8000/docs  

Миграции сейчас до **`012_cloud_pub`**.

Celery (отдельные терминалы, из `backend/orchestrator`):

```bash
celery -A app.tasks.celery_app worker --loglevel=info
celery -A app.tasks.celery_app beat --loglevel=info
```

Beat нужен для: авто-масштаб облака (30с), verify публикации (2ч), backup, escalations, queue sync.

## 5. GPU-воркер (локально)

### Без GPU (smoke)

```bash
cd worker
# .env.worker или экспорт:
set WORKER_PIPELINE_MODE=stub
set WORKER_TOKEN=worker-dev-token
set WORKER_ID=local-gpu-01
set ORCHESTRATOR_WS_URL=ws://localhost:8000/ws/worker
set MINIO_ENDPOINT=http://localhost:9010
set MINIO_ACCESS_KEY=minioadmin
set MINIO_SECRET_KEY=minioadmin
set REDIS_URL=redis://localhost:6382/0
set WORKER_DEPLOY=local

py -3 worker_agent.py
```

Или Docker stub:

```bash
docker build -t kwork-worker:stub --build-arg INSTALL_TRELLIS=0 worker
docker run --rm --env-file .env kwork-worker:stub
```

### С GPU + TRELLIS (приёмка ≤3 мин)

```bash
# образ с весами (долго при первом билде)
docker build -t kwork-worker:latest worker

# E2E на железе
set WORKER_PIPELINE_MODE=trellis
set WORKER_DEPLOY=local
set WORKER_E2E_BUDGET_LOCAL_SEC=180
py -3 worker/scripts/e2e_trellis_acceptance.py --photos path\to\12jpg --fail-on-budget
```

Облако Intelion/Immers:

```bash
set CLOUD_API_MOCK=1
py -3 worker/cloud/provision.py --action flavors --provider intelion
# или без mock + CLOUD_API_TOKEN:
py -3 worker/cloud/provision.py --action create --provider intelion --gpu rtx4090
```

Админка: Воркеры → «Облачный инстанс» / Start / Stop.

## 9. Типичные сбои

| Симптом | Что сделать |
|---------|-------------|
| `connection refused` Postgres | порт **5435**, не 5432; контейнер up |
| Redis ошибка | URL с портом **6382** |
| MinIO 403/connect | endpoint **9010** |
| Admin не пускает | `ADMIN_VPN_REQUIRED=false`, `ADMIN_2FA_REQUIRED=false` |
| Alembic | из `backend/orchestrator`, `.env` в корне репо подхватывается |
| Cloud API 502 | `CLOUD_API_MOCK=1` или валидный `CLOUD_API_TOKEN` |
| TRELLIS падает без GPU | `WORKER_PIPELINE_MODE=stub` или `TRELLIS_ALLOW_STUB_FALLBACK=1` только для smoke |
