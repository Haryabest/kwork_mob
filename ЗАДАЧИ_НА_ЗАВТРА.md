### 20.4 — Аватар профиля (§20.8.1)

**Сейчас:** в backend нет `avatar` — только email/ФИО/phone в `PATCH /user/me`.

**Сделать:**

1. **Миграция:** `users.avatar_url` (nullable string) или `avatar_key` в MinIO bucket `models` / отдельный `avatars`.
2. **API:**
   - `POST /user/me/avatar` — multipart, max ~2 MB, jpg/png/webp, resize на клиенте или сервере (Pillow если уже в проекте, иначе только validate MIME).
   - `DELETE /user/me/avatar`
   - В `GET /user/me` и `_user_payload` — поле `avatar_url` (presigned или public path).
3. **UI:** `apps/web-seller/src/app/settings/page.tsx` — вкладка «Профиль»: `Avatar` + upload + удалить; показ в `SellerShell` header вместо заглушки «3D».

**DoD:** загрузка → виден в settings и в шапке; удаление сбрасывает на дефолт.

### 20.6 — Страница сотрудника (§20.5.3)

**Сейчас:** `apps/web-seller/src/app/team/page.tsx` — таблица, действия inline. API уже есть:

- `GET /company/members/{user_id}/tasks`
- `GET /company/members/{member_id}/sessions`
- `POST /company/members/{member_id}/sessions/revoke`
- `PATCH .../role`, `PATCH .../limits`, `DELETE /members/{user_id}`

**Сделать:**

1. Новая страница `apps/web-seller/src/app/team/[userId]/page.tsx`.
2. Клик по строке в `/team` → переход на detail.
3. Блоки на странице:
   - Профиль: email, ФИО, роль, дата join, статус
   - Лимиты: concurrent orders, monthly spending, categories
   - Статистика: заказов / моделей / сумма (агрегат из tasks или отдельный endpoint если нет — посчитать на клиенте из tasks)
   - Таблица заказов сотрудника (из `/tasks`)
   - Лента audit (фильтр `user_id` в `GET /company/audit`)
   - Кнопки: изменить роль/лимиты (модалки как на team), revoke sessions, удалить
4. SubNav «← К команде» как на других team-страницах.

**DoD:** Owner открывает карточку любого member; все действия работают без возврата на общий список.
