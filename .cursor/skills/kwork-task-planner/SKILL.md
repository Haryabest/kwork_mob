# Kwork Task Planner

## Когда использовать

Пользователь просит: «новые задачи», «следующий срез», «P25», «что делать дальше».

## Источники

1. `СЛЕДУЮЩИЕ_ЗАДАЧИ.md` — текущий батч
2. `чтоготовосегодня.md` — что уже сделано (не дублировать)
3. `.claude/ТЗ.txt` — только prod scope по §

## Алгоритм

1. Прочитать `СЛЕДУЮЩИЕ_ЗАДАЧИ.md` и последние итерации в `чтоготовосегодня.md`.
2. Grep по репозиторию — убедиться, что задачи **не сделаны**.
3. Сформировать батч **3 задачи** (backend / web-seller / mobile) или ops отдельно.
4. Обновить `СЛЕДУЮЩИЕ_ЗАДАЧИ.md`: секция «Сделано в срезе P{N-1}» + новый «P{N}».

## Формат задач в СЛЕДУЮЩИЕ_ЗАДАЧИ.md

```markdown
## P{N} — backend
1. **Краткое имя** — что сделать §X.Y

## P{N} — web-seller
2. ...

## P{N} — mobile
3. ...
```

## Правила

- Только prod code, без ops в батче (ops — отдельный блок).
- Одна задача = один измеримый deliverable.
- Ссылаться на § ТЗ.
- Не предлагать уже закрытое в `чтоготовосегодня.md`.

## Выход

Кратко перечислить P{N} задачи пользователю; файл `СЛЕДУЮЩИЕ_ЗАДАЧИ.md` обновлён.

## В автономном цикле

Если активен скилл `kwork-p-loop` — после закрытия P{N} **сразу** сформировать P{N+1} в `СЛЕДУЮЩИЕ_ЗАДАЧИ.md` без вопроса пользователю.
