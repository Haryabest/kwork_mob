# Kwork P-Loop (автономный цикл)

## Один промпт пользователя

```
@.cursor/skills/kwork-p-loop

Авто P: начни с текущего батча в СЛЕДУЮЩИЕ_ЗАДАЧИ.md.
Цикл: verifier → coder → verifier → planner (P{N+1}) → commit push.
Повторяй P28, P29… пока в СЛЕДУЮЩИЕ_ЗАДАЧИ.md останутся только Ops.
Остановись и отчитайся.
```

Опционально: `начни с P27` / `сделай 3 батча` / `стоп после P30`.

## Связанные скиллы

| Шаг | Скилл |
|-----|-------|
| Проверка | `kwork-task-verifier` |
| Код | `kwork-task-coder` |
| Следующий батч | `kwork-task-planner` |

Читай их SKILL.md в начале цикла.

## Алгоритм одного батча P{N}

1. **Определить N** — из `СЛЕДУЮЩИЕ_ЗАДАЧИ.md` (секция `## P{N}`) или из промпта пользователя.
2. **Verifier (до кода)** — для каждой задачи P{N}: grep + `чтоготовосегодня.md`.
   - Все DONE → пропустить код, сразу planner P{N+1}.
   - NOT DONE / PARTIAL → кодить только недостающее.
3. **Coder** — реализовать по `kwork-task-coder`.
4. **Verifier (после кода)** — все задачи P{N} = DONE (файл/тест/OpenAPI).
5. **Документы** — Итерация в `чтоготовосегодня.md`; в `СЛЕДУЮЩИЕ_ЗАДАЧИ.md`: «Сделано P{N}» + секция **P{N+1}** (planner).
6. **Git** — `git commit` + `git push origin main`.
7. **Следующий батч** — N := N+1, вернуться к шагу 1.

## Условия остановки

Остановиться, когда **любое** из:

- В `СЛЕДУЮЩИЕ_ЗАДАЧИ.md` только блок `## Ops` (нет `## P{N}`).
- Пользователь указал лимит: «3 батча», «до P30».
- Verifier после 2 попыток не может закрыть задачу (PARTIAL) — отчитаться, не зацикливаться.
- Нет новых prod-задач в ТЗ (planner не может предложить P{N+1} без дублей).

## Planner внутри цикла

После каждого закрытого P{N}:

1. Grep — что из ТЗ ещё не в коде / не в `чтоготовосегодня.md`.
2. Записать **3–5 задач** в P{N+1}: backend + web-seller + mobile.
3. Не дублировать закрытое; ops — только в блок Ops.

## Отчёт после цикла

Краткая таблица:

| Батч | Статус | Commit |
|------|--------|--------|
| P27 | done | `abc1234` |
| P28 | skipped (уже было) | — |

И текущий `СЛЕДУЮЩИЕ_ЗАДАЧИ.md` (какой P следующий).

## Правила

- **Не спрашивать** «продолжить P28?» — продолжать, пока не сработает условие остановки.
- **Не останавливаться** между батчами без причины из «Условия остановки».
- Один commit на батч.
- PowerShell: `;` вместо `&&`.

## Примеры промптов

```
@.cursor/skills/kwork-p-loop авто P с P27
```

```
@.cursor/skills/kwork-p-loop сделай 2 батча и остановись
```

```
@.cursor/skills/kwork-p-loop
Проверь P27. Если готово — P28. Если нет — доделай P27, потом P28. Push после каждого.
```
